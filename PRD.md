# PRD — QuestLearn (학습자료 → 게임 퀘스트 퀴즈 변환기)

> 학습자료를 붙여넣으면 AI가 **5단계 게임 퀘스트 퀴즈**로 자동 변환해주는 웹 서비스

- 작성자: kjc
- 최초 작성일: 2026-07-22
- 버전: v1.0 (MVP)

---

## 1. 문제 정의 (Problem)

교육 실무자(교사·강사·교육 프로그램 운영자)는 수업 자료를 **복습/평가용 퀴즈**로 바꾸는 작업을 반복적으로 한다.

- 학습자료에서 핵심 개념을 뽑아 문제로 만드는 데 시간이 오래 걸린다.
- 단순 O/X·객관식 문제는 학습자의 몰입을 끌어내기 어렵다.
- 매번 수작업으로 만들다 보니 재사용·관리가 안 된다.

**→ 학습자료를 입력하면 AI가 자동으로 "게임 퀘스트형 퀴즈"로 변환하고, 저장·재사용·수정할 수 있게 하자.**

게임 퀘스트 형식(상황·문제·선택지·정답·해설·보상)은 학습자의 몰입도를 높이고, 교사는 그대로 수업에 활용할 수 있다.

## 2. 타겟 유저 (Target User)

- **주 사용자:** 초·중·고 교사, 학원 강사, 사내/공공 교육 프로그램 운영자
- **상황:** "이 학습자료로 오늘/이번 주 퀴즈를 만들어야 하는데 시간이 없다"
- **니즈:**
  - 자료만 넣으면 곧바로 쓸 수 있는 퀴즈가 나왔으면
  - 만든 퀴즈를 저장해두고 나중에 다시 꺼내 쓰고 싶다
  - 문제가 마음에 안 들면 직접 고칠 수 있어야 한다

## 3. 핵심 기능 (MVP)

과제 필수 기준(CRUD·DB·화면 2개+·배포·AI)을 모두 충족하도록 구성.

| # | 기능 | 설명 | CRUD |
|---|------|------|------|
| F1 | **퀘스트 생성** | 학습자료(제목+본문)를 입력 → Claude API가 5단계 게임 퀘스트 퀴즈 생성 → DB 저장 | Create |
| F2 | **퀘스트 목록** | 저장된 퀘스트를 카드 목록으로 조회 | Read |
| F3 | **퀘스트 상세/플레이** | 한 퀘스트를 열어 5단계 퀴즈를 순서대로 풀어봄(선택지 클릭→정답/해설 확인, 보상 획득) | Read |
| F4 | **퀘스트 삭제** | 필요 없는 퀘스트 삭제 | Delete |
| F5 | **퀘스트 제목 수정** | 저장된 퀘스트의 제목 수정 | Update |

- **AI 확장(F1):** Claude API를 사용해 학습자료 → 5단계 퀘스트 자동 생성. (과제의 "AI 기능 확장" 항목 충족)
- **게임 퀘스트 출력 규격:** 총 5단계, 각 단계는 `상황 설명 / 문제 / 선택지(4개) / 정답 / 해설 / 획득 보상` 포함, 한국어.

### 향후 확장 (Nice to have, MVP 이후)
- 로그인/회원가입(Supabase Auth)으로 유저별 퀴즈 관리
- 난이도·문항 수 조절 옵션
- 퀘스트 결과(점수) 저장, 학습자 공유 링크

## 4. 화면 구성 (Screens)

총 3개 화면 (과제 기준 "2개 이상" 충족).

1. **홈 / 생성 화면** (`/`)
   - 학습자료 제목 입력 + 본문 붙여넣기 textarea
   - "퀘스트 생성" 버튼 → 로딩 표시 → 생성 완료 시 상세 화면으로 이동
   - 하단에 최근 만든 퀘스트 목록 미리보기 링크

2. **퀘스트 목록 화면** (`/quests`)
   - 저장된 퀘스트 카드 리스트 (제목, 생성일, 미리보기)
   - 각 카드: [플레이] [제목수정] [삭제] 버튼

3. **퀘스트 상세/플레이 화면** (`/quests/[id]`)
   - 5단계 퀘스트를 순서대로 진행
   - 각 단계: 상황 설명 → 문제 → 선택지 클릭 → 정답/해설/보상 표시
   - 진행도 표시, 마지막 단계 후 완료 화면

### 상태/디테일
- 로딩 상태(스피너), 에러 처리(생성 실패 시 안내), 반응형(모바일 대응)

## 5. 데이터 구조 (Data Model)

DB: **Supabase (PostgreSQL)**

### 테이블: `quests`
| 컬럼 | 타입 | 설명 |
|------|------|------|
| `id` | uuid (PK, default gen_random_uuid()) | 퀘스트 고유 ID |
| `title` | text | 퀘스트 제목(학습자료 제목) |
| `source_material` | text | 입력한 학습자료 원문 |
| `stages` | jsonb | 5단계 퀘스트 데이터(배열) |
| `created_at` | timestamptz (default now()) | 생성 시각 |

### `stages` (jsonb) 구조 — 배열, 5개 원소
```json
[
  {
    "stage": 1,
    "situation": "상황 설명 (게임적 배경)",
    "question": "문제 텍스트",
    "choices": ["선택지1", "선택지2", "선택지3", "선택지4"],
    "answerIndex": 0,
    "explanation": "정답 해설",
    "reward": "획득 보상 (예: 지혜의 룬 +10)",
    "objective": "이 문제로 확인하는 핵심 학습목표 한 문장"
  }
]
```

## 6. API 설계 (Backend)

Next.js Route Handlers(`app/api/...`)로 구현. Supabase를 통해 DB 접근.

| 메서드 | 경로 | 설명 |
|--------|------|------|
| `POST` | `/api/quests` | 학습자료 받아 Claude로 퀘스트 생성 + DB 저장 (Create) |
| `GET` | `/api/quests` | 퀘스트 목록 조회 (Read) |
| `GET` | `/api/quests/[id]` | 퀘스트 단건 조회 (Read) |
| `PATCH` | `/api/quests/[id]` | 퀘스트 제목 수정 (Update) |
| `DELETE` | `/api/quests/[id]` | 퀘스트 삭제 (Delete) |

## 8. 검증 기준 (Acceptance)

- 해당 내용을 가르쳐 본 교사 1명이 "문제가 학습자료의 핵심 내용을 정확하게 반영하고 수업에서 사용할 수 있다"고 평가한다.
- 5단계 퀘스트가 각각 상황·문제·선택지·정답·해설·보상을 모두 포함한다.
- 배포된 URL로 누구나 접속해 생성·조회·수정·삭제가 동작한다.
